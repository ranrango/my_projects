# 工业界算法选型

更新时间：2026-05-29

## 1. 目标检测

推荐组合：

- PoC/边缘 CPU：YOLOv8n/YOLO11n ONNX。
- 边缘 GPU/Jetson：YOLO11s/m 或 RT-DETR 小模型，导出 TensorRT。
- 精度优先：YOLO11m/l、RT-DETR、DINO/DETR 系列，服务端部署。
- 小目标/高分辨率：切片推理 SAHI + YOLO/RT-DETR，或训练 4K/小目标增强模型。

工业建议：

- 不直接依赖 COCO 预训练模型上线。
- 至少采集现场白天、夜晚、逆光、遮挡、雨雾、不同摄像头角度的数据。
- 行人/车辆之外的工业类别应单独标注，例如 helmet、vest、forklift、fire、smoke、restricted_area_person。

## 2. 多目标跟踪

推荐组合：

- 标准实时跟踪：ByteTrack。
- 遮挡较多：BoT-SORT/OC-SORT。
- 跨镜头或长遮挡：检测 + ReID 外观特征。
- 人员属性/身份关联：单独训练 ReID 或属性识别模型。

工业建议：

- 只用 bbox IoU 的跟踪器适合简单场景。
- 人流密集、车流交叉、工厂遮挡场景应加入 ReID。
- 指标看 IDF1、ID Switch、MOTA，而不是只看画框效果。

## 3. 异常行为

推荐组合：

- 禁区闯入：检测 + ROI 多边形。
- 人群聚集：人检测 + 距离聚类 + 持续时间。
- 徘徊：轨迹半径/面积 + 时间窗。
- 跌倒：检测框规则只能做初筛；工业上线建议人体姿态估计 + 时序规则/轻量时序模型。
- PPE 合规：安全帽/反光衣检测，必要时用人框内二级检测。
- 烟火：专用 smoke/fire 检测模型 + 多帧确认。

工业建议：

- 规则要有持续时间、冷却时间和置信度分级。
- 报警最好输出证据：截图、短视频、目标框、track_id、ROI、时间戳。
- 高风险事件要接人工复核或二次模型确认，避免误报直接触发强动作。

## 4. 模型训练与闭环

推荐流程：

1. 现场视频抽帧。
2. 标注目标、事件、ROI。
3. 训练基线模型。
4. 端侧压测。
5. 收集误报/漏报。
6. 增量训练和版本发布。

最低验收指标建议：

- 核心目标 Recall 优先，通常要高于 0.9。
- 误报率按每路摄像头每天统计。
- 延迟按端到端事件时间统计，不只看单帧推理时间。
- 现场至少连续运行 7 天，统计断流、重连、内存增长和报警准确率。
