# ReID 行人重识别系统使用说明

本项目用于“行人重识别”：输入一张已经裁剪好的行人图片，在图库中检索最相似的行人图片。

简单理解：

```text
你的行人图片 -> 提取特征 -> 和图库图片比对 -> 返回最像的 Top-K 结果
```

注意：本项目不负责从整张监控画面里找人。如果你输入的是街景、监控原图、多人画面，需要先用检测模型把单个人裁剪出来。

## 1. 图片要求

输入图片建议满足：

- 图片里只有一个人。
- 人体尽量完整，最好包含头部、上身、下身。
- 不要只截脸、只截上半身、只截背包或局部。
- 人不要太小，裁剪后高度建议至少 `128px`，更推荐 `256px` 以上。
- 图片不要严重模糊、过暗、遮挡严重。
- 裁剪框最好接近“高瘦的人形框”。

模型内部会自动把图片缩放到：

```text
256 x 128
```

所以不用手动改尺寸。

## 2. 项目目录说明

常用文件：

```text
checkpoints/reid_baseline.pth       训练好的模型
deploy/toy_gallery_index.npz        示例 toy 图库索引
deploy/gallery_index.npz            你自己构建的图库索引
deploy/search_report.html           可视化检索报告
reid_baseline/deploy/build_gallery.py  构建图库索引
reid_baseline/deploy/search.py         命令行检索
reid_baseline/deploy/visualize_search.py 可视化检索
reid_baseline/deploy/api.py            API 服务
```

## 3. 安装依赖

进入项目目录：

```bash
cd /path/to/reid属性识别系统
```

安装依赖：

```bash
python3 -m pip install -r reid_baseline/requirements.txt
```

如果在云服务器上有 GPU，可以使用 `--device cuda`；本机没有 GPU 就用 `--device cpu`。

## 4. 用示例图库测试一张图片

把你的测试图片放到项目目录下，例如：

```text
my_query.jpg
```

生成可视化报告：

```bash
python3 -m reid_baseline.deploy.visualize_search \
  --checkpoint checkpoints/reid_baseline.pth \
  --index deploy/toy_gallery_index.npz \
  --query my_query.jpg \
  --topk 5 \
  --device cpu \
  --output deploy/my_search_report.html
```

然后打开：

```text
deploy/my_search_report.html
```

报告里会显示：

```text
左侧：你的 query 图片
右侧：Top-K 检索结果
每张结果图下方：rank、score、id、路径
```

注意：`toy_gallery_index.npz` 是示例小图库，只用于看流程，不代表真实效果。

## 5. 构建你自己的图库

如果你有一批候选行人图片，放到：

```text
gallery/
```

例如：

```text
gallery/
  person_001.jpg
  person_002.jpg
  camera_1/person_003.jpg
```

构建图库索引：

```bash
python3 -m reid_baseline.deploy.build_gallery \
  --checkpoint checkpoints/reid_baseline.pth \
  --gallery-dir gallery \
  --output deploy/my_gallery_index.npz \
  --batch-size 32 \
  --device cpu
```

如果是 GPU 环境：

```bash
python3 -m reid_baseline.deploy.build_gallery \
  --checkpoint checkpoints/reid_baseline.pth \
  --gallery-dir gallery \
  --output deploy/my_gallery_index.npz \
  --batch-size 128 \
  --device cuda
```

## 6. 在自己的图库里检索

命令行查看：

```bash
python3 -m reid_baseline.deploy.search \
  --checkpoint checkpoints/reid_baseline.pth \
  --index deploy/my_gallery_index.npz \
  --query my_query.jpg \
  --topk 5 \
  --device cpu
```

生成 HTML 效果图：

```bash
python3 -m reid_baseline.deploy.visualize_search \
  --checkpoint checkpoints/reid_baseline.pth \
  --index deploy/my_gallery_index.npz \
  --query my_query.jpg \
  --topk 10 \
  --device cpu \
  --output deploy/my_search_report.html
```

打开：

```text
deploy/my_search_report.html
```

## 7. 分数怎么看

返回结果里有 `score`：

```text
score 越高，越相似
```

粗略参考：

```text
0.80 以上：通常很像
0.65 - 0.80：建议人工复核
0.65 以下：通常不太可靠
```

这个阈值不是固定的，实际要根据你的摄像头、角度、清晰度、衣服变化来调。

## 8. 启动 API 服务

先准备模型和图库索引：

```text
checkpoints/reid_baseline.pth
deploy/my_gallery_index.npz
```

启动：

```bash
export REID_CHECKPOINT=checkpoints/reid_baseline.pth
export REID_INDEX=deploy/my_gallery_index.npz
export REID_DEVICE=cpu
export REID_API_KEY="<set-a-strong-random-token>"

uvicorn reid_baseline.deploy.api:app --host 0.0.0.0 --port 8000
```

GPU 环境把 `REID_DEVICE` 改成：

```bash
export REID_DEVICE=cuda
```

健康检查：

```bash
curl -H "X-API-Key: test-key" http://127.0.0.1:8000/health
```

上传图片检索：

```bash
curl -X POST "http://127.0.0.1:8000/search?topk=5" \
  -H "X-API-Key: test-key" \
  -F "file=@my_query.jpg"
```

返回示例：

```json
{
  "results": [
    {
      "rank": 1,
      "id": "person_001.jpg",
      "score": 0.8732
    }
  ],
  "latency_ms": 38.42,
  "threshold": null
}
```

## 9. 更新图库

如果图库图片变了，需要重新构建索引：

```bash
python3 -m reid_baseline.deploy.build_gallery \
  --checkpoint checkpoints/reid_baseline.pth \
  --gallery-dir gallery \
  --output deploy/my_gallery_index.npz \
  --device cpu
```

API 服务运行中可以热加载：

```bash
curl -X POST http://127.0.0.1:8000/reload \
  -H "X-API-Key: test-key"
```

## 10. 常见问题

### 结果不像怎么办？

优先检查：

- query 图片是不是单人裁剪图。
- 图库里是否真的有同一个人。
- 图片是否太模糊、太暗、遮挡严重。
- query 和图库是否衣服变化太大。

### 能不能输入整张监控画面？

不建议。应该先检测行人，再把单个人裁剪出来输入本系统。

### 能不能识别换衣服的人？

当前模型主要依赖外观和衣着，换衣服后可靠性会明显下降。

### 图库很大怎么办？

当前 Numpy 索引适合中小图库。几十万到百万级图库建议升级 FAISS 或向量数据库。

## 11. 推荐使用流程

```text
1. 准备训练好的模型 checkpoint
2. 准备图库行人裁剪图
3. 构建图库索引
4. 用 visualize_search.py 先人工看效果
5. 调整 score 阈值
6. 启动 API 服务
7. 接入业务系统
```
