# ReID 行人重识别算法与方法说明

本文档说明本项目使用到的主要算法、模型结构、训练方法、评估方法和部署检索方法。

## 1. 任务定义

行人重识别，Person Re-Identification，简称 ReID。

任务目标是：

```text
给定一张行人图片，在图库中找到同一个人的图片。
```

它不是目标检测任务。目标检测负责“找出图片里哪里有人”，ReID 负责“判断两张行人图是不是同一个人”。

完整业务流程通常是：

```text
视频/图片
  -> 行人检测
  -> 行人裁剪
  -> ReID 特征提取
  -> 图库相似度检索
  -> 返回 Top-K 结果
```

本项目主要实现其中的：

```text
行人裁剪图 -> ReID 特征提取 -> 相似度检索
```

## 2. 数据集

项目主要按 Market1501 数据集格式组织。

Market1501 常见目录：

```text
Market-1501-v15.09.15/
  bounding_box_train/   训练集
  query/                查询集
  bounding_box_test/    图库集 gallery
```

文件名示例：

```text
0001_c1s1_001051_00.jpg
```

含义：

```text
0001  人员 ID
c1    摄像头 ID
s1    序列 ID
后面是帧号/图片编号
```

训练时，本项目会把原始人员 ID 映射成连续标签：

```text
原始 PID -> 0, 1, 2, ..., num_classes-1
```

这是因为分类损失 `CrossEntropyLoss` 要求类别标签必须是连续整数。

评估和部署时，会过滤 Market1501 中的无效干扰 ID：

```text
pid <= 0
```

## 3. 图像预处理

输入图片会统一处理成：

```text
尺寸：256 x 128
颜色：RGB
归一化：ImageNet mean/std
```

归一化参数：

```text
mean = [0.485, 0.456, 0.406]
std  = [0.229, 0.224, 0.225]
```

训练阶段还使用了：

```text
RandomHorizontalFlip
```

也就是随机水平翻转，用于增强模型对左右方向变化的鲁棒性。

## 4. 模型结构

本项目使用 ResNet50 作为主干网络。

整体结构：

```text
输入行人图 3x256x128
  -> ResNet50 backbone
  -> Global Average Pooling
  -> Linear embedding layer
  -> BatchNorm
  -> Classifier
```

代码位置：

```text
reid_baseline/model/reid_model.py
```

输出包括两个部分：

```text
feat_bn: ReID 特征向量
logits: 训练分类输出
```

默认特征维度：

```text
512
```

部署时主要使用 `feat_bn` 作为行人特征。

## 5. Backbone：ResNet50

ResNet50 是残差网络，核心思想是使用残差连接：

```text
y = F(x) + x
```

残差连接能缓解深层网络训练中的梯度消失问题，使模型可以学习更强的视觉特征。

本项目使用 ResNet50 去掉最后的平均池化层和全连接分类层，只保留卷积特征提取部分：

```text
backbone = resnet50 去掉 avgpool 和 fc
```

然后接自己的 ReID embedding 层和分类头。

## 6. 特征层与 BNNeck 思想

模型中有一层：

```text
BatchNorm1d(embedding_dim)
```

这类似 ReID 中常见的 BNNeck 思路。

作用：

- 让特征分布更稳定。
- 分类训练更容易收敛。
- 检索时使用归一化后的 embedding 表示行人。

项目中返回的是：

```text
feat_bn
```

也就是经过 BatchNorm 的特征。

## 7. 训练损失

训练使用两个损失相加：

```text
Loss = CrossEntropyLoss + TripletLoss
```

### 7.1 CrossEntropyLoss

交叉熵损失用于身份分类。

模型把每张训练图片预测成一个人员 ID 类别：

```text
图片 -> 分类 logits -> 人员 ID
```

作用：

- 让不同身份在分类空间中可分。
- 提升模型对身份类别的辨别能力。

### 7.2 TripletLoss

Triplet Loss 用于度量学习。

目标是：

```text
同一个人的特征距离更近
不同人的特征距离更远
```

三元组包括：

```text
Anchor    锚点图片
Positive  同一个人的图片
Negative  不同人的图片
```

基本形式：

```text
max(0, d(anchor, positive) - d(anchor, negative) + margin)
```

本项目默认：

```text
margin = 0.3
```

代码位置：

```text
reid_baseline/loss/triplet_loss.py
```

## 8. 特征提取

部署时，输入一张行人图，模型输出一个 512 维特征向量。

之后会做 L2 归一化：

```text
feature = feature / ||feature||
```

这样后续可以直接用内积计算余弦相似度。

代码位置：

```text
reid_baseline/deploy/inferencer.py
```

## 9. 相似度计算

部署检索时使用余弦相似度：

```text
score = query_feature · gallery_feature
```

因为特征已经 L2 归一化，所以内积等价于余弦相似度。

分数越高，说明越相似。

粗略参考：

```text
score >= 0.80       通常很像
0.65 <= score < 0.80 建议人工复核
score < 0.65        通常不够可靠
```

实际阈值需要根据业务数据重新调。

## 10. Top-K 检索

图库中每张图片都会提前提取特征，并保存为索引：

```text
deploy/gallery_index.npz
```

索引包含：

```text
features  图库特征矩阵
paths     图库图片路径
ids       业务 ID
metadata  索引元信息
```

检索流程：

```text
query 图片
  -> 提取 query feature
  -> 与图库 features 矩阵相乘
  -> 得到每张图库图的 score
  -> 取 Top-K
```

为了提升性能，本项目使用：

```text
numpy.argpartition
```

它比完整排序更适合只取 Top-K 的场景。

## 11. 评估指标

项目评估使用 ReID 常见指标：

```text
mAP
Rank-1
Rank-5
Rank-10
```

代码位置：

```text
reid_baseline/inference/evaluate_reid.py
```

### 11.1 Rank-k

Rank-k 表示：

```text
正确匹配是否出现在前 k 个结果中
```

例如：

```text
Rank-1 = 0.7957
```

表示大约 79.57% 的 query，第一名就是正确身份。

### 11.2 mAP

mAP，全称 mean Average Precision。

它不仅关心有没有找对，还关心所有正确结果在排序中的整体位置。

mAP 越高，说明检索排序整体越好。

## 12. 本项目已跑出的结果

在 Market1501 上，使用 ResNet50 预训练权重训练 10 轮后，评估结果为：

```text
mAP:    0.5588
Rank-1: 0.7957
Rank-5: 0.9121
Rank-10: 0.9362
```

这说明模型已经能学到有效的行人外观特征，但还不是最强 ReID 方案。

## 13. API 部署方法

部署时主要使用 FastAPI。

服务接口：

```text
GET  /health   健康检查
POST /search   上传 query 图片并检索
POST /reload   热加载图库索引
```

API 支持：

- API Key 鉴权
- 上传大小限制
- Top-K 限制
- score 阈值过滤
- 默认隐藏服务器绝对路径
- 热加载图库索引

## 14. TorchScript 导出

项目支持导出 TorchScript：

```text
deploy/reid_feature_extractor.pt
```

用途：

- 非 Python 服务加载模型
- 单独部署特征提取模块
- 和其他工程系统集成

导出后模型输入：

```text
[N, 3, 256, 128]
```

输出：

```text
[N, 512]
```

## 15. 当前方法的优点

- 结构清晰，训练、评估、部署流程完整。
- 使用 ResNet50，稳定可靠，容易复现。
- CrossEntropy + TripletLoss 是经典 ReID 训练组合。
- 支持命令行检索、HTML 可视化、API 服务和 TorchScript 导出。
- 当前 Numpy 检索适合中小规模图库。

## 16. 当前方法的局限性

- 主要依赖衣着和外观，换衣服后可靠性会下降。
- 对严重模糊、遮挡、低光照图片不够稳定。
- 输入必须是行人裁剪图，不适合直接输入整张监控画面。
- 当前图库索引是 Numpy 矩阵，超大图库建议升级 FAISS。
- 没有加入重排序 re-ranking，mAP 还有提升空间。
- 没有使用更先进的 Transformer ReID 模型。

## 17. 可继续优化方向

后续可以继续提升：

```text
1. 加入 Random Erasing 数据增强
2. 使用更好的 Triplet 挖掘策略
3. 加入 re-ranking 后处理
4. 使用 OSNet、TransReID 等更强模型
5. 使用 FAISS 提升大图库检索速度
6. 增加检测 + 跟踪模块，形成完整视频 ReID 系统
7. 增加前端页面，支持上传 query、展示 Top-K
8. 增加日志、监控、批量检索和图库管理
```

## 18. 总结

本项目采用经典深度学习 ReID 流程：

```text
ResNet50 特征提取
  + CrossEntropyLoss 身份分类
  + TripletLoss 度量学习
  + L2 归一化 embedding
  + 余弦相似度 Top-K 检索
```

它已经具备训练、评估、部署和可视化检索能力，适合作为行人重识别项目的工程化基线。
